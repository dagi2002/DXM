# SDK Integration Guide

DXM Pulse tracks user behavior through a lightweight JavaScript SDK that you add to any website with a single script tag. The SDK automatically captures page views, clicks, scrolls, navigation events, and Core Web Vitals.

---

## Quick Start

Add this script tag to the `<head>` of every page you want to track:

```html
<script
  src="https://cdn.dxmpulse.et/sdk/dxm.js"
  data-site-id="YOUR_SITE_KEY"
  async
></script>
```

Replace `YOUR_SITE_KEY` with the site key shown in your DXM Pulse dashboard under **Settings → Sites**.

That's it. The SDK starts collecting data immediately.

---

## Finding Your Site Key

1. Log into the DXM Pulse dashboard
2. Go to **Settings** in the left sidebar
3. Under **Sites**, find your website
4. Copy the **Site Key** (format: `site_xxxxxxxx`)

---

## What Gets Collected Automatically

| Event | Description |
|---|---|
| `pageview` | Fires on initial page load and on SPA route changes |
| `click` | Every user click — captures coordinates and CSS selector of target |
| `scroll` | Tracks scroll depth as a percentage of page height |
| `navigation` | SPA route changes (pushState / popState) |
| `vital` | Core Web Vitals: LCP, FID, CLS, TTFB, FCP |

No personally identifiable information (PII) is collected by default. Text content of elements and form values are never captured.

---

## Optional: Identify Users

If your app has user accounts, you can associate sessions with a user ID:

```javascript
window.dxm?.identify('user_12345');
// or with additional traits:
window.dxm?.identify('user_12345', {
  name: 'Abebe Kebede',
  plan: 'pro'
});
```

This sets `user_id_external` on the session record. PII like email addresses should not be passed — use an internal ID.

---

## Optional: Track Custom Events

```javascript
window.dxm?.track('upgrade_clicked', { plan: 'pro', source: 'banner' });
window.dxm?.track('video_played', { video_id: 'intro_tour' });
```

Custom events appear in the event stream associated with the current session.

---

## Single-Page Application (SPA) Support

The SDK automatically listens to `popstate` and patches `history.pushState` to detect client-side navigation. No configuration needed for React Router, Vue Router, Next.js, or other SPA frameworks.

Each route change fires a `navigation` event and updates the `url` field on subsequent events.

---

## Session Recording

The SDK also captures a full DOM event log for session replay using **rrweb**. This allows you to watch back exactly what a user did, including mouse movement, clicks, and DOM changes.

Session replays are sent to `POST /collect` alongside behavioral events and stored in the `session_replays` table.

**Privacy note:** The rrweb recorder is configured to mask all `<input>` and `<textarea>` values automatically. Form data is never recorded.

---

## How Events Are Sent

Events are batched in memory and flushed every **3 seconds** to `POST /collect`:

```
POST https://api.dxmpulse.et/collect
Content-Type: application/json

{
  "siteKey": "site_abc123",
  "sessionId": "sess_xyz789",   ← UUID generated by SDK, stored in sessionStorage
  "events": [ ... ],
  "meta": {
    "userAgent": "...",
    "language": "am",
    "screenWidth": 1440,
    "screenHeight": 900
  }
}
```

The flush also happens on `visibilitychange` (tab close / switch) to capture the session end.

---

## Data Retention and Privacy

- Sessions and events are stored for the duration of your plan's retention window
- No cookies are set on the visitor's browser by the SDK — `sessionId` is stored in `sessionStorage` only (cleared when tab closes)
- GDPR / privacy: DXM Pulse collects behavioral data only. You are responsible for disclosing this collection in your privacy policy

---

## Self-Hosting the SDK

If you're running DXM Pulse on your own infrastructure, serve `dxm.js` from your own CDN or server and update `SDK_CDN_URL` / `VITE_SDK_CDN_URL` in your `.env` to match.

The SDK source is located at `apps/web/public/sdk/dxm.js`.

---

## Verifying Installation

After adding the script tag:

1. Open your website in a browser
2. Open DevTools → Network tab
3. Filter by `collect`
4. You should see `POST /collect` requests firing every ~3 seconds

In the DXM Pulse dashboard, open **Session Replays** — your session should appear within seconds of visiting your site.

---

## Troubleshooting

**No data appearing in dashboard**

- Check the browser console for errors — likely a wrong site key or CORS issue
- Verify `data-site-id` matches exactly what's shown in Settings
- Check that `WEB_ORIGIN` in your API `.env` matches the origin of the instrumented site

**`405 Method Not Allowed` on /collect**

- The API server is not running or the URL in the SDK is wrong

**Sessions show as bounced even for active users**

- The session is marked bounced if it has only 1 `pageview` event and no interactions. Ensure the SDK script tag is loading correctly (check for script errors blocking execution)
