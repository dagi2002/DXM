# Database Schema

DXM Pulse uses **SQLite** via the `better-sqlite3` driver. The database file is stored at `apps/api/data/dxm.db`.

Run migrations with:
```bash
npm run migrate -w apps/api
```

Migrations are **idempotent** (safe to re-run) — all DDL uses `IF NOT EXISTS`.

---

## Entity Relationship Diagram

```
workspaces
    │
    ├── users (N)
    ├── sites (N)
    │       │
    │       ├── sessions (N)
    │       │       │
    │       │       ├── events (N)
    │       │       └── session_replays (1)
    │       │
    │       └── funnels (N)
    │
    └── alerts (N)
```

## Tables

### `workspaces`

The top-level tenant. Every piece of data belongs to a workspace.

| Column | Type | Description |
|---|---|---|
| `id` | TEXT PK | UUID |
| `name` | TEXT | Company or product name |
| `plan` | TEXT | `free` \| `starter` \| `pro` |
| `billing_status` | TEXT | `active` \| `past_due` \| `cancelled` |
| `telegram_chat_id` | TEXT | Optional — Telegram chat for alerts |
| `telegram_bot_token` | TEXT | Optional — workspace-specific bot token |
| `digest_enabled` | INTEGER | `0` or `1` — enables weekly digest delivery |
| `digest_language` | TEXT | `en` \| `am` |
| `chapa_customer_id` | TEXT | Optional — Chapa customer reference |
| `created_at` | DATETIME | Auto-set on insert |

---

### `users`

Team members who can log into the DXM Pulse dashboard.

| Column | Type | Description |
|---|---|---|
| `id` | TEXT PK | UUID |
| `workspace_id` | TEXT FK | References `workspaces.id` (CASCADE DELETE) |
| `name` | TEXT | Display name |
| `email` | TEXT UNIQUE | Login email |
| `password_hash` | TEXT | bcrypt hash |
| `role` | TEXT | `owner` \| `admin` \| `viewer` |
| `avatar` | TEXT | Optional URL |
| `last_login` | DATETIME | Updated on each successful login |
| `refresh_token_hash` | TEXT | bcrypt hash of current refresh token (rotated on use) |
| `created_at` | DATETIME | Auto-set on insert |

**Indexes:** `idx_users_email` on `(email)`

---

### `sites`

A website being tracked by DXM Pulse. One workspace can have multiple sites.

| Column | Type | Description |
|---|---|---|
| `id` | TEXT PK | UUID |
| `workspace_id` | TEXT FK | References `workspaces.id` (CASCADE DELETE) |
| `name` | TEXT | Human-readable site name (e.g. "Marketing Site") |
| `domain` | TEXT | Base domain (e.g. `example.com`) |
| `site_key` | TEXT UNIQUE | The value passed as `data-site-id` in the SDK script tag |
| `created_at` | DATETIME | Auto-set on insert |

**Indexes:** `idx_sites_key` on `(site_key)`

---

### `sessions`

A single user visit to a tracked site. Created or updated on each event batch.

| Column | Type | Description |
|---|---|---|
| `id` | TEXT PK | UUID generated by the SDK |
| `site_id` | TEXT FK | References `sites.id` |
| `workspace_id` | TEXT FK | References `workspaces.id` (denormalized for fast queries) |
| `started_at` | DATETIME | Timestamp of first event |
| `ended_at` | DATETIME | Timestamp of last event |
| `duration` | INTEGER | Session length in seconds |
| `user_agent` | TEXT | Raw browser user agent string |
| `device` | TEXT | `desktop` \| `mobile` \| `tablet` |
| `browser` | TEXT | Browser name (e.g. `Chrome`) |
| `language` | TEXT | Browser language (e.g. `en`, `am`) |
| `screen_width` | INTEGER | Viewport width in px |
| `screen_height` | INTEGER | Viewport height in px |
| `entry_url` | TEXT | First URL visited in this session |
| `user_id_external` | TEXT | Optional — set by `identify()` SDK call |
| `clicks` | INTEGER | Total click count |
| `scroll_depth` | INTEGER | Max scroll depth percentage (0–100) |
| `total_events` | INTEGER | Total event count |
| `bounced` | INTEGER | `1` if user left without interacting |
| `converted` | INTEGER | `1` if a goal event was recorded |
| `completed` | INTEGER | `1` if session ended cleanly |
| `created_at` | DATETIME | Auto-set on insert |
| `updated_at` | DATETIME | Updated on each event batch |

**Indexes:** `idx_sessions_workspace`, `idx_sessions_site`, `idx_sessions_created`

---

### `events`

Individual interaction events within a session.

| Column | Type | Description |
|---|---|---|
| `id` | INTEGER PK | Auto-increment |
| `session_id` | TEXT FK | References `sessions.id` (CASCADE DELETE) |
| `type` | TEXT | `pageview` \| `click` \| `scroll` \| `navigation` \| `vital` \| `custom` \| `identify` |
| `ts` | INTEGER | Epoch milliseconds (from SDK, not server time) |
| `x` | INTEGER | Click/scroll X coordinate |
| `y` | INTEGER | Click/scroll Y coordinate |
| `scroll_depth` | INTEGER | Scroll depth % at time of event |
| `target` | TEXT | CSS selector of clicked element |
| `url` | TEXT | Page URL at time of event |
| `value_text` | TEXT | Vital value, custom event name, or identify userId |
| `created_at` | DATETIME | Server insert time |

**Event type details:**

| Type | `x`/`y` | `target` | `value_text` |
|---|---|---|---|
| `pageview` | — | — | — |
| `click` | Click coords | CSS selector | — |
| `scroll` | — | — | — |
| `scroll_depth` set on session | | | |
| `navigation` | — | — | Destination URL |
| `vital` | — | Metric name (e.g. `LCP`) | Value in ms (as string) |
| `custom` | — | — | Event name |
| `identify` | — | — | External user ID |

**Indexes:** `idx_events_session`, `idx_events_type`

---

### `session_replays`

Replay summary row for session replay playback. One row per session.

| Column | Type | Description |
|---|---|---|
| `session_id` | TEXT PK | References `sessions.id` (CASCADE DELETE) |
| `events_json` | TEXT | Legacy fallback replay blob; new writes keep this as `[]` |
| `size_bytes` | INTEGER | Total replay payload size across stored chunks |
| `created_at` | DATETIME | Auto-set on insert |

---

### `session_replay_chunks`

Chunked rrweb event storage for replay ingestion and playback. Multiple rows per session.

| Column | Type | Description |
|---|---|---|
| `session_id` | TEXT PK/FK | References `sessions.id` (CASCADE DELETE) |
| `chunk_index` | INTEGER PK | Replay chunk order within the session |
| `events_json` | TEXT | JSON array for just this replay chunk |
| `size_bytes` | INTEGER | Byte length of this chunk payload |
| `created_at` | DATETIME | Insert/update timestamp |

---

### `alerts`

Auto-detected or manually created alerts for a workspace.

| Column | Type | Description |
|---|---|---|
| `id` | TEXT PK | UUID |
| `workspace_id` | TEXT FK | References `workspaces.id` (CASCADE DELETE) |
| `site_id` | TEXT FK | References `sites.id` (nullable) |
| `type` | TEXT | `error` \| `performance` \| `frustration` \| `conversion` |
| `severity` | TEXT | `low` \| `medium` \| `high` \| `critical` |
| `title` | TEXT | Short description |
| `description` | TEXT | Longer detail |
| `resolved` | INTEGER | `0` = open, `1` = resolved |
| `affected_sessions` | INTEGER | Count of affected sessions |
| `telegram_sent` | INTEGER | `1` if Telegram notification was sent |
| `created_at` | DATETIME | Auto-set on insert |
| `resolved_at` | DATETIME | Set when resolved |

**Indexes:** `idx_alerts_workspace`, `idx_alerts_resolved`

---

### `funnels`

Saved conversion funnels with ordered URL-pattern steps.

| Column | Type | Description |
|---|---|---|
| `id` | TEXT PK | UUID |
| `workspace_id` | TEXT FK | References `workspaces.id` (CASCADE DELETE) |
| `site_id` | TEXT FK | References `sites.id` (nullable) |
| `name` | TEXT | Funnel name (e.g. "Signup Flow") |
| `steps_json` | TEXT | JSON array: `[{ name: string, urlPattern: string }]` |
| `created_at` | DATETIME | Auto-set on insert |

**`steps_json` format:**
```json
[
  { "name": "Landing Page", "urlPattern": "/" },
  { "name": "Pricing", "urlPattern": "/pricing" },
  { "name": "Signup", "urlPattern": "/signup" },
  { "name": "Dashboard", "urlPattern": "/dashboard" }
]
```

The `urlPattern` is matched against `events.url` using SQL `LIKE '%pattern%'`.

## Indexes Summary

| Index | Table | Columns | Purpose |
|---|---|---|---|
| `idx_sessions_workspace` | sessions | workspace_id | Filter sessions by workspace |
| `idx_sessions_site` | sessions | site_id | Filter sessions by site |
| `idx_sessions_created` | sessions | created_at | Time-range queries |
| `idx_events_session` | events | session_id | Join events to session |
| `idx_events_type` | events | type | Filter by event type |
| `idx_alerts_workspace` | alerts | workspace_id | Filter alerts by workspace |
| `idx_alerts_resolved` | alerts | resolved | Filter open vs closed alerts |
| `idx_users_email` | users | email | Login lookup |
| `idx_sites_key` | sites | site_key | SDK event validation |
